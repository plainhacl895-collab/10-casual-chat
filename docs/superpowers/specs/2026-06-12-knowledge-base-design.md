# 房产AI知识库系统设计

## 背景

用户（佳佳）是一线房产经纪人，同时自学 AI 技术。当前学习方式是碎片化的（刷视频、看文章、收藏链接），缺少系统性和条理性。参考了 Xuan酱（Codex+Obsidian）和金岛主（Claude Code+Obsidian）两个知识库搭建教程后，决定搭建自己的 AI 驱动知识库。

## 目标

- 把碎片化输入变成结构化积累，覆盖 **房产业务**（房源产品/谈判/销售）和 **AI 技术** 两大领域
- 现阶段为主：自己学、自己查、自己复习
- 未来可扩展：输出给客户/团队

## 当前状态

- Claude Code：已通过 VS Code 插件安装，日常使用中
- Obsidian：未安装
- 电脑空间：D盘 161G 可用，E盘 171G 可用
- 学习习惯：碎片时间为主，每天约 1 小时固定时段

## 方案三：混合阶梯

### 阶段一（立即启动）：极简三层

**目标**：建仓库 + 通 AI + 开始囤内容，2 周跑通最小闭环。

#### 文件夹结构

```
e:\房产AI知识库\
├── 0-收集箱/              # 一切原始素材丢这里
│   ├── 临时收藏/           # 随手存的链接、截图、视频
│   └── 待整理/             # 标记了"这个有用"但还没提炼的
│
├── 1-整理中/              # AI帮你消化后的笔记
│   ├── 房产业务/
│   │   ├── 房源产品/       # 户型、价格、地段、竞品分析
│   │   ├── 谈判/           # 话术、策略、案例
│   │   └── 销售/           # 拓客、跟进、转化、客户心理
│   └── AI技术/
│
├── 2-沉淀/                # 提炼成可复用的方法论/模板
│
└── _配置/                 # Claude Code 配置文件
    ├── CLAUDE.md           # 主入口
    ├── profile.md          # 我是谁 + 业务方向
    ├── rules.md            # 整理标准 + 笔记格式
    ├── tasks.md            # 每日/每周任务
    ├── 结构映射.md          # 当前完整文件夹结构
    └── 标签映射.md          # 标签 → 目录映射表
```

#### 设计原则

- **数字前缀**：`0-` `1-` `2-` 强制排序，天然引导信息从左到右流动
- **宽度优先于深度**：一级只放这几个大类，阶段一不往下细分——细分是过早设计
- **0-收集箱不分类**：碎片时间丢进去即可，分类是 AI 的事
- **\_配置** 放最后：下划线排序到最后，不碍眼

#### 工具连接方式

Obsidian 和 Claude Code 看同一个文件夹，文件夹本身就是桥接器：

```
e:\房产AI知识库\
    └── 同一个文件夹 ← Obsidian 开它为 Vault，VS Code Claude Code 开它为工作区
```

- **Obsidian**：浏览、编辑、随手记录
- **VS Code Claude Code**：AI 读、写、整理、定时任务
- 不需要 Cloudy 插件做桥接

**_配置/profile.md**：

```markdown
- 业务核心三块：房源产品、谈判、销售
- 同时自学 AI 技术，偏实操落地
```

**_配置/rules.md**：
```markdown
# 整理标准与笔记格式

## 房产业务整理维度
- 房源产品：关键参数（户型/面积/价格段）、卖点、适合客群、竞品对比
- 谈判：具体话术、对方出招→我方应对话术、心理博弈逻辑
- 销售：拓客渠道、跟进节奏、逼单时机、客户分类方法

## 笔记格式
- 顶部标注：来源、时间、标签
- 关键结论用 ### 标题突出
- 我的思考用 > 引用块标注

## 不做的
- 不过度分类，阶段一保持简洁
- 不帮我写我没要的东西
- 不清楚的地方主动问
```

**_配置/tasks.md**：
```markdown

## 每日
1. 检查 0-收集箱 是否有新内容，有就帮我：
```

### 阶段二（运行 2 周后）：引入自动化和 Skill 沉淀

触发条件：收集箱累计超过 50 条内容，且阶段一流程跑通无卡点。

#### 新增能力

1. **定时蒸馏**：Claude Code 自动化任务，每日/每周定时整理知识库内容
2. **Skill 沉淀**：把反复出现的工作模式提炼为可复用的 Skill（例如：房源评估 skill、客户分析 skill）
3. **文件夹自然演化**：根据实际使用情况调整子目录结构

#### 暂不引入

- Karpathy 四层架构的"回流"环节（阶段三再评估）
- PPT/视频等输出功能

## 接口设计

三条核心接口，保证后续改动不被锁死。

### 1. 文件夹：只增不删

- 一级结构 `0-收集箱/` `1-整理中/` `2-沉淀/` `_配置/` 为稳定基础，不改不删
- 后续只在现有目录内加子目录，不改一级结构
- `结构映射.md` 记录当前完整结构，AI 在需要导航时读该文件

### 2. 配置：模块化

- 每个配置单文件职责单一，改业务方向改 profile.md，改整理标准改 rules.md，不用在主文件里翻找
- CLAUDE.md 只做入口引用，不写细节

### 3. 标签：用映射表

- 标签不硬编码在 rules.md 中，全部在 `标签映射.md` 维护
- 新增业务线（如 #招聘 #管理）只需加一行映射

---

## 不做的

- 不装 Cloudy 插件（不想用 Claude Code 命令行界面）
- 不一开始就搭完美四层架构（先跑通三层再说）
- 不放在 C 盘、D 盘项目目录（放 E 盘独立目录）
- 不在阶段一做过细的二级分类
